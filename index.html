Execution model

Three-phase transaction engine.

Every debugging session follows a deterministic, reversible pipeline. No magic. No surprises.

PHASE 01 / READ

Observe

Safely gather system state. Pod logs, network interfaces, DNS records, resource metrics. Zero mutations, full observability across every layer.

PHASE 02 / ANALYZE

Reason

DocLM reasons across collected data. Correlates symptoms to root cause with multi-step inference spanning all infra domains simultaneously.

PHASE 03 / MODIFY

Act

User-confirmed changes with full state snapshots. Automatic LIFO rollback on failure. Every modification is reversible by design.
